# 小龙虾交接说明

## 交接目标

请小龙虾把本包作为 15 分钟监控策略后的“交易决策核心”。

本包只做判断和订单参数生成，不负责保存密钥，不负责签名，不负责真实发送 API 请求。

## 接入步骤

1. 每 15 分钟收完 M15 K 线后运行一次。
2. 给每个币种传入最新已收盘的 `15m` 和 `1h` K 线。
3. 调用 `trading_engine.evaluate_live_cycle()`。
4. 根据返回的 `action` 字段执行。
5. 如果返回 `RISK_PAUSE`，当天不再开新仓，只允许处理已有平仓动作。

## 执行动作

`OPEN_BINANCE_OPTION`：

- 币安期权买入。
- 只做非 BTC。
- 每张数量固定 `0.01`。

`CLOSE_BINANCE_OPTION`：

- 币安期权卖出平仓。
- 触发原因可能是权利金 3 倍、RE_DENSE 盈利、EMA 盈利保护、时间退出。

`OPEN_OKX_SWAP`：

- 欧易永续合约开仓。
- BTC 名义金额 `6U`。
- 其他币名义金额 `5U`。

`CLOSE_OKX_SWAP`：

- 欧易永续合约平仓。
- 触发原因可能是 TP、HARD_SL、EMA_REV、RE_DENSE。

`NO_TRADE`：

- 无交易机会。

`RISK_PAUSE`：

- 风控暂停。
- 不要开新仓。

## 小龙虾需要自己处理

- API Key 和 Secret。
- Binance/OKX 请求签名。
- OKX 合约张数最小单位校验。
- 下单失败重试和报警。
- 成交后同步真实持仓。
- 每日 0 点重置今日盈亏统计。

## 最重要的实盘限制

请不要把未收盘 K 线喂给策略。  
本策略所有信号都默认在 M15 收盘后产生，并在下一根 M15 开盘执行。
